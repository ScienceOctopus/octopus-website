Reproducability Ratings for Blogposts

January 1, 0001

Reproducability Ratings

The top and bottom of all my blogposts shows a rating for the reproducibility of the content with the following considerations:

  • Is the author citable?
  • Is the data citable?
  • Is the code citable (and replicable)?

For the ratings I’ll use the following Font Awesome icons:

Not reproducible!

Complete lack of unique identifier or programmatic access to resource.

As of the first implementation of these ratings (early February 2018), I simply include raw HTML into the blogpost. As blogdown Hugo templates automatically create summaries from the first X characters of the post, this messed up my blog summaries a lot. Thankfully, Yihui has a blogpost on modifying hugo post summaries that was fairly easy to implement. To be explicit, I introduced the following code:

<p class="intro">
{{ with .Description }}
{{ $.Scratch.Set "summary" (markdownify .) }}
{{ else }}
{{ $.Scratch.Set "summary" ((delimit (findRE "(<p.*?>(.|\n)*?</p>\\s*)+" .Content) "[…] ") | plainify | truncate (default 200 .Site.Params.summary_length) (default " …" .Site.Params.text.truncated ) | replaceRE "&" "&" | safeHTML) }}
{{ end }}
{{ $.Scratch.Get "summary" }}
</p>

… into these two locations, in place of <p class="intro">{{ .Summary }}</p>

# layouts/_default/single.html
# layouts/partial/recent_posts.html

Please note that these instructions are only valid for the devcows/hugo-universal-theme that my blog was using at the time of writing in February 2018. However, it’s a massive improvement over the default theme as now I can write completely custom click-bait summaries for my blogs, and include emojis r emo::ji("rabbit"), by adding this to the YAML of a post :
